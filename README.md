# Parser API for n8n & Telegram Bot

API для высокопроизводительного парсинга данных с сайтов автозапчастей. Оптимизировано для работы с n8n и Telegram ботами.

## Основные возможности
- **Site Discovery**: Узнайте список доступных сайтов через API.
- **Concurrency Control**: Использование семафора (лимит 15 страниц) для предотвращения перегрузки сервера.
- **Batch Processing**: Одновременный сбор данных по деталям товаров.
- **n8n Ready**: Чистый JSON формат без лишнего оформления.

---

## Эндпоинты

### 1. Получить список сайтов
**`GET /sites`**

Возвращает список идентификаторов сайтов, которые поддерживает парсер.

**Ответ:**
```json
{
  "sites": ["skladmotorov.ru", "leoparts.ru", "euroauto.ru"]
}
```

---

### 2. Поиск (POST) — Рекомендуемый
**`POST /search`**

Основной метод для поиска артикулов.

**Тело запроса (JSON):**
| Поле | Тип | Обязательно | Описание |
| :--- | :--- | :--- | :--- |
| `queries` | `list[str]` | Да | Список артикулов или поисковых запросов. |
| `sites` | `list[str]` | Нет | Список сайтов для поиска. Если не указано — поиск идет по всем. |

**Пример запроса:**
```json
{
  "queries": ["06A103383AS", "03G103383"],
  "sites": ["skladmotorov.ru"]
}
```

---

### 3. Поиск (GET)
**`GET /search?query={query}&sites={sites}`**

**Параметры:**
- `query`: Строка запросов через запятую.
- `sites`: (Опционально) Список сайтов через запятую.

**Пример:** `/search?query=06A103383AS,03G103383&sites=skladmotorov.ru`

---

### 4. Проверка состояния
**`GET /health`**

Используется для мониторинга контейнера. Возвращает `{"status": "ok"}`.

---

## Формат данных в ответе

Все методы поиска возвращают объект с ключом `results`, который имеет иерархическую структуру: **Сайт -> Артикул -> Список товаров**.

### Пример ответа (JSON):
```json
{
  "results": {
    "skladmotorov.ru": {
      "06A103383AS": [
        {
          "name": "ГБЦ 1.6 BSE VW Golf V / Passat B6",
          "price": "15 000 руб.",
          "cross_numbers": "06A103383AS, 06A103383BM",
          "url": "https://skladmotorov.ru/autopart-product/123",
          "searched_query": "06A103383AS",
          "site": "skladmotorov.ru"
        }
      ],
      "другой_артикул": []
    },
    "другой_сайт.com": {
      "06A103383AS": [...]
    }
  }
}
```

### Структура данных:
- **`результаты` (dict)**: Ключи — это домены сайтов.
- **`артикул` (dict)**: Внутри каждого сайта ключи — это исходные поисковые запросы.
- **`список товаров` (list)**: Массив найденных совпадений. Если ничего не найдено, там будет сообщение о пустом результате или пустой список (в зависимости от конкретного парсера).

### Поля товара:
- **`searched_query`**: Артикул, по которому был найден этот товар.
- **`price`**: Строка с ценой.
- **`cross_numbers`**: Строка со списком кросс-номеров.
- **`site`**: Домен сайта.

## Развертывание
### Docker
```bash
docker build -t auto-parser .
docker run -p 8000:8000 auto-parser
```
API будет доступно на порту `8000`.
